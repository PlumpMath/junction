accept_publish(service, mask, value, method, handler=None, schedule=False)¶

Set a handler for incoming publish messages

Parameters:
  • service (anything hash-able) – the incoming message must have this service
  • mask (int) – value to be bitwise-and’ed against the incoming id, the result of which must mask the ‘value’ param
  • value (int) – the result of routing_id & mask must match this in order to trigger the handler
  • method (string) – the method name
  • handler (callable) – the function that will be called on incoming matching messages
  • schedule (bool) – whether to schedule a separate greenlet running handler for each matching message. default False.
Raises :
  • ImpossibleSubscription if there is no routing ID which could possibly match the mask/value pair
  • OverlappingSubscription if a prior publish registration that overlaps with this one (there is a service/method/routing id that would match both this and a previously-made registration).

rpc(service, routing_id, method, args=None, kwargs=None, timeout=None, singular=False)¶

Send an RPC request and return the corresponding response

This will block waiting until the response has been received.

Parameters:
  • service (anything hash-able) – the service name (the routing top level)
  • routing_id (int) – The id used for routing within the registered handlers of the service.
  • method (string) – the method name to call
  • args (tuple) – the positional arguments to send along with the request
  • kwargs (dict) – keyword arguments to send along with the request
  • timeout (float or None) – maximum time to wait for a response in seconds. with None, there is no timeout.
  • singular (bool) – if True, only send the request to a single peer
Returns:

a list of the objects returned by the RPC’s targets. these could be of any serializable type.
Raises :
  • Unroutable if no peers are registered to receive the message
  • WaitTimeout if a timeout was provided and it expires

Wait for the response for any (the first) of multiple futures

This method will block until a response has been received.

Parameters:

  • timeout (float or None) – maximum time to wait for a response in seconds. with None, there is no timeout.
Returns:
