connect()¶

Initiate the connection to a proxying node

publish(service, method, routing_id, args, kwargs)¶

Send a 1-way message

Parameters:
  • service (anything hash-able) – the service name (the routing top level)
  • method (string) – the method name to call
  • routing_id (int) – The id used for routing within the registered handlers of the service.
  • args (tuple) – the positional arguments to send along with the request
  • kwargs (dict) – keyword arguments to send along with the request
Returns:

None. use ‘rpc’ methods for requests with responses.
Raises :

Unroutable if the client doesn’t have a connection to a node
publish_receiver_count(service, method, routing_id, timeout=None)¶

Get the number of peers that would handle a particular publish

This method will block until a response arrives

Parameters:
  • service (anything hash-able) – the service name
  • method (string) – the method name
  • routing_id (int) – the id used for narrowing within the (service, method) handlers
  • timeout (int, float or None) – maximum time to wait for the response
Raises :
  • Unroutable if no peers are registered to receive the message
  • WaitTimeout if a timeout was provided and it expires
rpc(service, method, routing_id, args, kwargs, timeout=None)¶

Send an RPC request and return the corresponding response

This will block waiting until the response has been received.

Parameters:
  • service (anything hash-able) – the service name (the routing top level)
  • method (string) – the method name to call
  • routing_id (int) – The id used for routing within the registered handlers of the service.
  • args (tuple) – the positional arguments to send along with the request
  • kwargs (dict) – keyword arguments to send along with the request
  • timeout (float or None) – maximum time to wait for a response in seconds. with None, there is no timeout.
Returns:

a list of the objects returned by the RPC’s targets. these could be of any serializable type.
Raises :
  • Unroutable if no peers are registered to receive the message
  • WaitTimeout if a timeout was provided and it expires

wait_any(futures, timeout=None)¶

Wait for the response for any (the first) of multiple futures

This method will block until a response has been received.

Parameters:
  • futures (list) – a list made up of rpc rpc and dependent objects on which to wait (only for the first one)
  • timeout (float or None) – maximum time to wait for a response in seconds. with None, there is no timeout.
Returns:

one of the RPCs or Dependents from futures – the first one to be completed (or any of the ones that were already completed) for which completed is True.
Raises :
  • WaitTimeout if a timeout was provided and it expires
